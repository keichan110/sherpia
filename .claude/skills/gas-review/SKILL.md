直近のgit変更を対象に、gas-reviewerサブエージェントを使ってGAS互換性レビューを実施する。

## ステップ1: レビュー対象ファイルの特定

ユーザーがファイルを明示した場合はそれを対象にする。指定がない場合は以下の順で変更ファイルを探す：

```bash
# 1. 未コミットの変更（ステージ済み＋未ステージ）
git diff --name-only HEAD

# 2. なければブランチの変更（mainとの差分）
git diff --name-only main...HEAD
```

`src/` 以下の `.ts` ファイルのみを対象とする。対象が見つからない場合は「レビュー対象の src/*.ts ファイルが見つかりません」と伝えて終了する。

## ステップ2: gas-reviewerサブエージェントでレビュー

対象ファイルが確定したら `gas-reviewer` サブエージェントを起動し、以下を伝える：

- レビュー対象ファイルのリスト（パス一覧）
- 各ファイルを Read ツールで読んでレビューすること

## ステップ3: 結果の報告

gas-reviewer の出力をそのままユーザーに報告する。
